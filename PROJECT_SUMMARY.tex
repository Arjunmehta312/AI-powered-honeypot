% Section 1: Overview
\section*{\colorbox{lightgray}{\parbox{\linewidth}{\textbf{1. Project Overview}}}}

This project implements an intelligent honeypot analysis system that automatically detects, classifies, and analyzes cyber attacks using machine learning. It processes real-world attack data from AWS EC2 honeypots deployed in London, achieving \textbf{99.99--100\% accuracy} in threat classification.

\textbf{Key Capabilities:}
\begin{itemize}[leftmargin=*,itemsep=0pt,topsep=2pt]
    \item Real-time attack classification (8 attack types)
    \item Threat severity scoring (Low/Medium/High)
    \item Protocol identification (10+ network protocols)
    \item Geographic attack mapping and visualization
    \item Behavioral pattern analysis
    \item Interactive web-based dashboard
\end{itemize}

% Section 2: Dataset
\section*{\colorbox{lightgray}{\parbox{\linewidth}{\textbf{2. Dataset \& Statistics}}}}

\textbf{Source:} AWS EC2 Honeypot --- London Region \\
\textbf{Collection Period:} May 1--5, 2021 (5 days) \\
\textbf{Total Records:} 70,327 attack attempts \\
\textbf{Unique Attackers:} 3,508 IP addresses \\
\textbf{Unique Ports:} 30,337 targeted ports

\textbf{Attack Type Distribution:}
\begin{itemize}[leftmargin=*,itemsep=0pt,topsep=2pt]
    \item HTTP Exploits: 23.1\% (16,272 records)
    \item TLS/SSL Attacks: 16.1\% (11,344 records)
    \item RDP Attacks: 15.2\% (10,721 records)
    \item Port Scans: 9.3\% (6,571 records)
    \item SMB Attacks: 4.8\% (3,368 records)
    \item SSH Brute Force: 1.3\% (947 records)
    \item Database Attacks: 0.8\% (536 records)
    \item Unknown: 29.2\% (20,568 records)
\end{itemize}

% Section 4: Engineered Features
\section*{\colorbox{lightgray}{\parbox{\linewidth}{\textbf{4. Feature Engineering (19 Features)}}}}

\textbf{Payload Features:} Length, malicious keyword detection, type encoding

\textbf{Interaction Features:} Port-hour combinations, IP frequency-payload interactions, weekend-night patterns

\textbf{Reputation Features:} IP reputation scores based on attack history

% Section 7: How It Works
\section*{\colorbox{lightgray}{\parbox{\linewidth}{\textbf{7. System Workflow}}}}

\textbf{Step 1: Data Ingestion} \\
Raw honeypot logs are loaded from CSV format containing timestamps, source IPs, ports, payloads, and geographic data.

\textbf{Step 2: Preprocessing} \\
Data is cleaned, duplicates removed, missing values handled, timestamps parsed, and protocols identified using signature matching.

\textbf{Step 3: Feature Extraction} \\
Behavioral features are engineered including payload characteristics, temporal patterns, network metrics, and IP reputation scores.

\textbf{Step 4: Attack Classification} \\
ML models predict attack types based on 19 engineered features, outputting classification with confidence scores.

\textbf{Step 5: Threat Scoring} \\
Separate models assign severity levels (Low/Medium/High) based on attack characteristics and historical patterns.

\textbf{Step 6: Visualization} \\
Results are displayed in interactive dashboard with filtering, drill-down capabilities, and real-time prediction interface.
